# ウェイクワードモデル改善の検討

## 現状（v1モデル）の評価

| 項目 | 内容 |
|------|------|
| 打率 | 約30%（実用不可） |
| 暴発 | 多い（テレビ音などに反応） |
| 学習データ | ポジティブ50件・ネガティブ240件 |
| 学習時間 | 数分（RTX5080） |

### なぜ精度が出ないか

1. **声の多様性不足**: edge-ttsの無料音声2種類のみ（Nanami / Keita）。実際のカマロさん本人の声には対応していない
2. **ネガティブが薄い**: 日本語20フレーズ+ホワイトノイズだけ。テレビ・音楽・環境音を学習していない
3. **データ量が少なすぎる**: ML業界の感覚では学習データが2桁少ない（通常は数千〜数十万件）
4. **データ拡張なし**: ピッチ・速度・ノイズ重畳などの augmentation を入れていない
5. **モデルが単純**: 軽量CNN。時系列モデリング（LSTM/GRU/Transformer）がない

---

## 改善の方向性（段階別）

### Phase 1: 既存パイプラインの強化（半日〜1日）

**やること:**
- TTS声の追加: 有料TTS（Google Cloud TTS / Azure TTS）で日本語10声以上
- レート・ピッチ・音量のバリエーション拡大（5パターン → 30パターン）
- ノイズ重畳: 各サンプルにホワイトノイズ・ピンクノイズを様々なSNRで混入
- ネガティブを 240 → 2,000件 に増やす
- 「クロード」と紛らわしい単語を意図的に追加（「コード」「クラウド」「クラブ」「群衆」など）

**期待効果:** 打率60〜70%、暴発半減

**コスト:** Google TTSの場合 約500円程度

---

### Phase 2: 大規模日本語データセット導入（2〜3日）

**やること:**
- **Common Voice Japanese** (Mozillaオープンソース、約30GB)
  - 数万人の実音声サンプル
  - ランダム1万件をネガティブに使用
- **JVS corpus** (100人の日本語話者、無料)
  - 多様な日本人の声でネガティブを補強
- **MUSAN dataset** (音楽・スピーチ・ノイズ)
  - 環境音学習用

**期待効果:** 打率80%、暴発を1日数回程度に抑制

**コスト:** ストレージ50GB必要、ダウンロード時間数時間

---

### Phase 3: OpenWakeWordフルパイプライン（1週間）

**やること:**
- OpenWakeWord公式の学習パイプラインを採用
- **Piper TTS** で日本語の多数話者を合成（数千〜数万声）
- **Room Impulse Response (RIR)** 畳み込みで様々な部屋の音響を再現
- 音声データ拡張: SpecAugment / TimeMask / FreqMask
- ResNet系のbackbone + LSTM時系列モデル
- 数百万〜数千万サンプルで学習（RTX5080なら一晩）

**期待効果:** 打率90%以上、暴発ほぼゼロ

**コスト:** 学習時間8〜24時間、ディスク100GB

**参考リンク:** https://github.com/dscripka/openWakeWord/blob/main/docs/training.md

---

### Phase 4: ユーザー特化型ファインチューニング（運用後）

**やること:**
- 実運用で集めたカマロさん本人の音声を保存
- 1〜2週間で50〜100サンプル蓄積される想定
- それらを **positive サンプル** として追加学習
- スピーカー特化のverifier層を追加（OpenWakeWordの `train_custom_verifier`）

**期待効果:** カマロさんの声に特化して打率99%、他人や他の音には反応しなくなる

**コスト:** 自動収集の仕組みを実装する手間（半日）

---

## カマロさん本人の声を録音する方法（強力な裏技）

実音声録音はTTSより遥かに精度が上がります。

**準備:**
1. スマホやPCのマイクで「ねえクロード」と50〜100回録音
2. 様々な口調・距離・声量・気分（眠い時/起きてる時）で
3. WAV形式で `data/positive_real/` に保存

**効果:** これだけで打率が一気に80〜90%まで上がる可能性あり。Phase 2や3より費用対効果が高い。

## 代替案との比較

### Porcupine（審査待ち）
- 現在審査中。商用利用前提のため通らない可能性あり
- 通れば即最高品質（Phase 3相当）
- カマロさんが他のことに集中できる

### Whisper（現状）
- 打率高めだがCPU負荷が継続的にかかる
- 寝室静音環境なら実用十分
- 「クロード」単体の暴発リスクは残る

### 自作モデル（このドキュメント）
- ローカル完結、外部依存なし
- 精度はPhaseに応じて段階的に上がる
- 本人音声を入れれば最高クラスにできる

---

## 結論

**まずは Phase 1 + 本人音声100件（ルートA）から試すのがおすすめ。**

- 半日で完成
- 失敗してもWhisperに戻せばよい
- 本人音声100件は今晩でも録音可能
- うまくいけば Phase 2/3 へ進む価値あり

Phase 3のフルパイプラインは魅力的ですが、まず軽量で結果を見てから判断する方が無駄が少ないです。
